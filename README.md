# Algoritmia y Programación para Ingeniería de Sistemas

Curso académico diseñado para estudiantes de primer semestre del Programa de Ingeniería de Sistemas de la Universidad del Pacífico.

Este repositorio contiene guías teóricas, ejemplos explicados paso a paso, ejercicios prácticos, talleres y proyectos orientados al aprendizaje de la algoritmia, el pensamiento computacional y la programación básica utilizando pseudocódigo, PSeInt y Java.

---

## Descripción del Curso

La algoritmia constituye una de las bases fundamentales en la formación de todo ingeniero de sistemas, porque permite desarrollar la capacidad de analizar problemas, descomponerlos en partes más simples y construir soluciones ordenadas mediante instrucciones lógicas.

Este curso ha sido diseñado para estudiantes que inician su proceso de formación en programación. Por esta razón, los temas se desarrollan de manera progresiva, comenzando con conceptos básicos como algoritmo, variable, dato, entrada, proceso y salida, hasta llegar a la construcción de programas en Java aplicando estructuras de control, funciones, arreglos, matrices, algoritmos de búsqueda, algoritmos de ordenamiento y principios básicos de Programación Orientada a Objetos.

El enfoque principal del curso es que el estudiante aprenda primero a **pensar la solución** antes de escribir código. Para ello, cada problema será abordado desde el análisis, el diseño del algoritmo, la representación en pseudocódigo, la prueba de escritorio y finalmente su implementación en Java.

## Metodología de Trabajo

Cada unidad temática seguirá una estructura didáctica orientada al aprendizaje progresivo:

1. **Introducción conceptual:** explicación inicial del tema.
2. **Teoría básica:** presentación de los conceptos fundamentales.
3. **Análisis del problema:** identificación de entradas, procesos y salidas.
4. **Diseño del algoritmo:** solución en lenguaje natural o pseudocódigo.
5. **Prueba de escritorio:** validación manual del algoritmo.
6. **Implementación en Java:** traducción de la lógica a código.
7. **Explicación línea a línea:** análisis detallado de las instrucciones principales.
8. **Ejercicios resueltos:** problemas desarrollados con solución completa.
9. **Ejercicios propuestos:** actividades para practicar de forma independiente.
10. **Retos de aplicación:** problemas integradores de mayor nivel.
11. **Taller evaluativo:** actividad para evidenciar el aprendizaje.

El objetivo no es memorizar código, sino comprender cómo se construye una solución lógica, ordenada y funcional.

## Contenido General del Curso

### Unidad 1. Introducción a la Algoritmia

En esta unidad se estudian los conceptos iniciales que permiten comprender qué es programar y por qué los algoritmos son importantes.

**Temas:**

* Concepto de algoritmo.
* Concepto de programa.
* Computadores y lenguajes de programación.
* Pensamiento computacional.
* Análisis de problemas.
* Entrada, proceso y salida.
* Enfoque sistémico en la solución de problemas.
* Contrato de un problema: objetivo, datos de entrada, proceso, salida y pruebas.

**Resultado esperado:**

El estudiante reconoce la importancia de analizar un problema antes de programarlo y comprende la estructura básica de una solución algorítmica.

* Variables.
* Constantes.
* Identificadores.
* Palabras reservadas.
* Tipos de datos.
* Datos enteros, reales, caracteres, cadenas y booleanos.
* Operadores aritméticos.

* Operadores lógicos.

**Resultado esperado:**

El estudiante utiliza correctamente variables, tipos de datos y operadores para construir expresiones algorítmicas y programas básicos.

## Ejemplo de Ruta para Resolver un Problema

Antes de escribir código, cada problema debe trabajarse siguiendo esta estructura:

### 1. Enunciado

¿Qué problema se desea resolver?

### 2. Análisis

¿Qué datos se necesitan?

**Entradas:**

Datos que el usuario debe ingresar.

**Proceso:**

Operaciones o decisiones necesarias para resolver el problema.

**Salidas:**

Resultado que debe mostrar el programa.

### 3. Diseño del algoritmo

Representación de la solución mediante lenguaje natural, pseudocódigo o diagrama de flujo.

### 4. Prueba de escritorio

Validación manual del algoritmo usando datos de prueba.

### 5. Implementación

Traducción del algoritmo a Java.

### 6. Verificación

Ejecución del programa y revisión de resultados.
