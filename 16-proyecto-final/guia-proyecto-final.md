# Guía del Proyecto Final

## Curso: Algoritmia y Programación

## Presentación

El proyecto final tiene como propósito integrar los temas trabajados durante el curso mediante el desarrollo de una aplicación en Java.

El estudiante debe demostrar que puede analizar un problema, diseñar una solución, escribir código organizado, aplicar estructuras de control, usar arreglos o listas, crear métodos y presentar el proyecto en GitHub.

---

## 1. Objetivo general

Desarrollar una aplicación en Java que solucione un problema sencillo mediante el uso de los conceptos fundamentales de programación vistos en el curso.

---

## 2. Temas que debe integrar el proyecto

El proyecto debe incluir, según su nivel de complejidad:

- Variables y tipos de datos.
- Operadores.
- Condicionales.
- Ciclos.
- Métodos.
- Arreglos o `ArrayList`.
- Búsqueda.
- Validación de datos.
- Programación Orientada a Objetos.
- Git y GitHub.
- Documentación en Markdown.

---

## 3. Modalidad de trabajo

El proyecto puede desarrollarse:

- Individual.
- En parejas.
- En equipos pequeños, si el docente lo autoriza.

Cada integrante debe poder explicar el código.

---

## 4. Requisitos mínimos del proyecto

El proyecto debe cumplir con estos requisitos:

1. Estar desarrollado en Java.
2. Tener una estructura ordenada.
3. Usar al menos dos clases propias.
4. Usar métodos para organizar la lógica.
5. Usar condicionales.
6. Usar ciclos.
7. Usar arreglos o `ArrayList`.
8. Validar datos importantes.
9. Tener menú de opciones.
10. Estar subido a GitHub.
11. Tener un archivo `README.md`.
12. Incluir evidencias de ejecución.

---

## 5. Estructura sugerida del proyecto

```text
proyecto-final-java/
│
├── README.md
├── src/
│   ├── Principal.java
│   ├── Estudiante.java
```

---

## 6. Etapas de desarrollo

### Etapa 1. Selección del problema

El equipo debe elegir un problema sencillo y concreto.

Ejemplos:

- Sistema de notas.
- Sistema de inventario.
- Sistema de biblioteca.
- Control de gastos.
- Agenda de tareas.
- Registro de ventas.

---

### Etapa 2. Análisis

Antes de programar, se debe definir:

| Elemento | Pregunta |
|---|---|
| Problema | ¿Qué se quiere resolver? |
| Usuarios | ¿Quién usará el programa? |
| Entradas | ¿Qué datos se necesitan? |
| Procesos | ¿Qué cálculos o decisiones se harán? |
| Salidas | ¿Qué resultados mostrará? |

---

### Etapa 3. Diseño

El estudiante debe definir:

- Clases necesarias.
- Atributos de cada clase.
- Métodos principales.
- Menú de opciones.
- Validaciones.
- Flujo general del programa.

---

### Etapa 4. Implementación

En esta etapa se escribe el código Java.

Recomendaciones:

- Programar por partes.
- Probar cada método.
- No dejar todo para el final.
- Usar nombres claros.
- Hacer commits frecuentes.

---

### Etapa 5. Pruebas

El proyecto debe probarse con diferentes datos.

Ejemplo para sistema de notas:

- Nota válida.
- Nota inválida.
- Estudiante aprobado.
- Estudiante reprobado.
- Lista vacía.
- Búsqueda existente.
- Búsqueda inexistente.

---

### Etapa 6. Documentación

El repositorio debe incluir un `README.md` con:

- Nombre del proyecto.
- Descripción.
- Funcionalidades.
- Tecnologías usadas.
- Cómo ejecutar.
- Autor o integrantes.
- Evidencias.

---

### Etapa 7. Entrega en GitHub

El proyecto debe estar en un repositorio de GitHub con commits visibles.

---

## 7. Funcionalidades mínimas sugeridas

El proyecto debe tener al menos un menú como este:

```text
===== MENÚ PRINCIPAL =====
1. Registrar información
2. Mostrar información
3. Buscar información
4. Actualizar información
5. Eliminar información
6. Mostrar resumen
7. Salir
```

No todos los proyectos deben tener exactamente las mismas opciones, pero sí deben mostrar una lógica completa.

---

## 8. Recomendaciones de programación

- Usar `Scanner` correctamente.
- Validar entradas.
- Usar `ArrayList` si la cantidad de datos puede cambiar.
- Crear métodos para no repetir código.
- Usar clases para representar entidades.
- Mantener el `main` lo más limpio posible.
- Probar después de cada cambio.
- No copiar código sin entenderlo.

---

## 9. Entregables

El estudiante debe entregar:

1. Enlace del repositorio de GitHub.
2. Código fuente completo.
3. README.md.
4. Documento breve de análisis.
5. Evidencias de ejecución.
6. Explicación del proyecto.
7. Sustentación oral si el docente la solicita.

---

## 10. Sustentación

Durante la sustentación, el estudiante debe poder explicar:

- Qué problema resuelve.
- Qué clases creó.
- Qué hace cada método importante.
- Cómo usa condicionales y ciclos.
- Cómo usa arreglos o ArrayList.
- Cómo valida datos.
- Cómo subió el proyecto a GitHub.
- Qué dificultades tuvo y cómo las solucionó.

---

## 11. Recomendación final

El proyecto final no busca que el estudiante haga una aplicación perfecta, sino que demuestre que comprende la lógica de programación y puede construir una solución organizada en Java.
