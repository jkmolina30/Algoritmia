# Entrega del Proyecto Final en GitHub

## Aclaración

El proyecto final se evaluará desde la programación estructurada en Java.  
No se exigirá Programación Orientada a Objetos.

---

## Estructura recomendada

```text
proyecto-final/
│
├── README.md
├── src/
│   └── Principal.java
├── docs/
│   ├── analisis.md
│   ├── pruebas.md
│   └── evidencias.md
└── recursos/
    └── imagenes/
```

---

## README obligatorio

El archivo `README.md` debe incluir:

- Nombre del proyecto.
- Descripción.
- Funcionalidades.
- Temas aplicados.
- Tecnologías utilizadas.
- Cómo ejecutar.
- Autor o integrantes.

---

## Temas aplicados

El README puede indicar que el proyecto usa:

- Java.
- Variables.
- Condicionales.
- Ciclos.
- Métodos.
- Arreglos o `ArrayList`.
- Validaciones.
- Git y GitHub.

---

- [ ] El proyecto compila.

- [ ] Tiene evidencias.
- [ ] Está subido a GitHub.
