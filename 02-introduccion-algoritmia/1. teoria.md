# Unidad 2. Introducción a la Algoritmia

## Presentación de la unidad

La algoritmia es una de las bases más importantes para aprender programación. Antes de escribir código en Java, es necesario aprender a pensar de manera ordenada, lógica y precisa.

Un computador no resuelve problemas por sí solo. El computador ejecuta instrucciones. Por eso, el programador debe ser capaz de analizar un problema y convertirlo en una secuencia de pasos claros. En esta unidad se estudiará qué es un algoritmo, cómo se analiza un problema, cómo se identifican entradas, procesos y salidas, y cómo se puede representar una solución antes de programarla.

---

## 1. ¿Qué es la algoritmia?
La algoritmia es el área que estudia el diseño, análisis y construcción de algoritmos. En palabras sencillas, la algoritmia nos enseña a resolver problemas mediante pasos ordenados. No se trata únicamente de escribir código. Se trata de aprender a pensar la solución.

---

## 2. ¿Qué es un algoritmo?
Un algoritmo es una secuencia ordenada y finita de pasos que permite resolver un problema o realizar una tarea.

Ejemplo cotidiano

Problema: preparar una limonada.

Algoritmo:

1. Tomar un vaso.
2. Agregar agua.
3. Cortar un limón.
4. Exprimir el limón.
5. Agregar azúcar.
6. Mezclar.
7. Servir.

Si los pasos se siguen correctamente, se obtiene una limonada.

En programación sucede algo similar: se define una serie de pasos para que el computador pueda resolver un problema.

---

## 3. Características de un algoritmo
Un algoritmo debe tener algunas características importantes.

| Característica | Explicación                                 |
| -------------- | ------------------------------------------- |
| Preciso        | Cada paso debe estar claramente definido    |
| Ordenado       | Los pasos deben seguir una secuencia lógica |
| Finito         | Debe terminar en algún momento              |
| Claro          | Debe poder entenderse sin ambigüedad        |
| Efectivo       | Debe resolver el problema planteado         |

---

## 4. Ejemplo de algoritmo sencillo

Problema: sumar dos números.

Algoritmo en lenguaje natural
1. Pedir el primer número.
2. Pedir el segundo número.
3. Sumar los dos números.
4. Mostrar el resultado.

**Análisis**

| Elemento | Descripción                    |
| -------- | ------------------------------ |
| Entrada  | Primer número y segundo número |
| Proceso  | Sumar los dos números          |
| Salida   | Resultado de la suma           |

---

## 5. Entrada, proceso y salida
Todo problema algorítmico se puede analizar usando tres elementos principales:


Entrada → Proceso → Salida

---

### 5.1. Entrada
La entrada corresponde a los datos que el algoritmo necesita para resolver el problema.

Ejemplos:

- Edad de una persona.
- Nombre de un estudiante.
- Tres notas académicas.
- Precio de un producto.
- Cantidad de horas trabajadas.

---

### 5.2. Proceso
El proceso corresponde a las operaciones, cálculos o decisiones que se realizan con los datos de entrada.

Ejemplos:

- Sumar dos números.
- Calcular un promedio.
- Multiplicar precio por cantidad.
- Convertir grados Celsius a Fahrenheit.
- Calcular el área de una figura.

---

### 5.3. Salida
La salida corresponde al resultado que el algoritmo debe mostrar.

Ejemplos:

- El promedio final.
- El valor total a pagar.
- El área calculada.
- Un mensaje de resultado.
- Una conversión de unidades.

---

## 6. Importancia de analizar antes de programar
Uno de los errores más comunes al aprender programación es empezar a escribir código sin entender bien el problema. Antes de programar, se debe responder:

- ¿Qué me están pidiendo?
- ¿Qué datos necesito?
- ¿Qué operaciones debo realizar?
- ¿Qué resultado debo mostrar?
- ¿Qué restricciones tiene el problema?
- ¿Cómo puedo comprobar si la respuesta es correcta?

Programar sin analizar es como construir una casa sin plano.

## 7. Lenguaje natural, pseudocódigo y código
Una solución puede representarse de varias formas.

| Representación   | Descripción                                                        |
| ---------------- | ------------------------------------------------------------------ |
| Lenguaje natural | Explicación con palabras comunes                                   |
| Pseudocódigo     | Representación estructurada parecida a un lenguaje de programación |
| Código           | Implementación en un lenguaje real, como Java                      |

---

## 8. Lenguaje natural
El lenguaje natural consiste en explicar los pasos de una solución usando palabras comunes.

Ejemplo

Problema: calcular el doble de un número.

Solución en lenguaje natural:

1. Pedir un número.
2. Multiplicar el número por 2.
3. Mostrar el resultado.

Este tipo de descripción es fácil de entender, pero no siempre es suficientemente precisa para un computador.

---

## 9. Pseudocódigo
El pseudocódigo es una forma de escribir algoritmos usando instrucciones similares a un lenguaje de programación, pero más cercanas al lenguaje humano.

Ejemplo:
Algoritmo CalcularDoble
    Definir numero, doble Como Entero

    Escribir "Ingrese un número:"
    Leer numero

    doble <- numero * 2

    Escribir "El doble es: ", doble
FinAlgoritmo

El pseudocódigo ayuda a organizar la lógica antes de escribir el programa en Java.

---

## 10. Código en Java
Después de diseñar el algoritmo, se puede implementar en Java.

Ejemplo:

import java.util.Scanner;

public class CalcularDoble {
    public static void main(String[] args) {
        Scanner entrada = new Scanner(System.in);

        int numero;
        int doble;

        System.out.print("Ingrese un número: ");
        numero = entrada.nextInt();

        doble = numero * 2;

        System.out.println("El doble es: " + doble);

        entrada.close();
    }
}

---

## 11. Diferencia entre algoritmo y programa

| Concepto  | Explicación                                      |
| --------- | ------------------------------------------------ |
| Algoritmo | Pasos lógicos para resolver un problema          |
| Programa  | Algoritmo escrito en un lenguaje de programación |


Un algoritmo puede existir en papel, en pseudocódigo o en lenguaje natural. Un programa es una versión ejecutable o implementada del algoritmo.

---

## 12. Enfoque sistémico en un problema algorítmico
El enfoque sistémico consiste en ver un problema como un sistema compuesto por entradas, procesos, salidas y posibles condiciones. En lugar de mirar el problema de forma desordenada, se analiza como una estructura.

Ejemplo

Problema: calcular el salario semanal de un trabajador.

| Elemento    | Descripción                                       |
| ----------- | ------------------------------------------------- |
| Entrada     | Horas trabajadas y valor por hora                 |
| Proceso     | Multiplicar horas trabajadas por valor por hora   |
| Salida      | Salario semanal                                   |
| Restricción | Las horas y el valor por hora deben ser positivos |

---

## 13. Contrato de un problema algorítmico
Un contrato es una forma ordenada de definir qué debe hacer una solución.

El contrato puede incluir:

| Elemento | Pregunta que responde            |
| -------- | -------------------------------- |
| Objetivo | ¿Qué debe resolver el algoritmo? |
| Entradas | ¿Qué datos necesita?             |
| Proceso  | ¿Qué operaciones realiza?        |
| Salidas  | ¿Qué resultado muestra?          |
| Pruebas  | ¿Cómo verifico que funciona?     |

---

## 14. Ejemplo de contrato algorítmico

Problema: calcular el área de un triángulo.

| Elemento | Descripción                                    |
| -------- | ---------------------------------------------- |
| Objetivo | Calcular el área de un triángulo               |
| Entradas | Base y altura                                  |
| Proceso  | Multiplicar base por altura y dividir entre 2  |
| Salida   | Área del triángulo                             |
| Prueba   | Si base = 10 y altura = 4, el área debe ser 20 |

---

## 15. Prueba de escritorio
La prueba de escritorio consiste en ejecutar manualmente un algoritmo usando datos de ejemplo. Sirve para revisar si la lógica es correcta antes de escribir el código en Java.

Ejemplo
Algoritmo: sumar dos números.

Datos de prueba:

num1 = 8
num2 = 5

| Paso              | num1 | num2 | suma |
| ----------------- | ---: | ---: | ---: |
| Inicio            |    - |    - |    - |
| Leer num1         |    8 |    - |    - |
| Leer num2         |    8 |    5 |    - |
| Calcular suma     |    8 |    5 |   13 |
| Mostrar resultado |    8 |    5 |   13 |


Resultado esperado:

La suma es 13

---

## 16. Errores comunes al diseñar algoritmos
Al iniciar, es normal cometer errores. Algunos de los más frecuentes son:

| Error                            | Ejemplo                                      |
| -------------------------------- | -------------------------------------------- |
| No identificar bien las entradas | Pedir solo una nota cuando se necesitan tres |
| Confundir proceso con salida     | Decir “mostrar promedio” como proceso        |
| Omitir pasos importantes         | No guardar el resultado de una operación     |
| No probar la solución            | Asumir que funciona sin verificar            |
| Usar nombres poco claros         | Usar `x`, `y`, `z` sin necesidad             |
| Escribir pasos desordenados      | Mostrar el resultado antes de calcularlo     |

---

## 17. Buenas prácticas iniciales
Desde el inicio del curso se recomienda:

- Leer el enunciado más de una vez.
- Subrayar los datos importantes.
- Identificar entradas, procesos y salidas.
- Escribir primero en lenguaje natural.
- Pasar luego a pseudocódigo.
- Realizar prueba de escritorio.
- Implementar en Java solo cuando la lógica esté clara.
- Usar nombres de variables claros.
- Probar con varios datos.

## 19. Ejemplo usando la plantilla

---

## 20. Resumen de la unidad
La algoritmia permite resolver problemas mediante pasos ordenados. Antes de programar en Java, es necesario analizar el problema, identificar entradas, procesos y salidas, diseñar una solución en lenguaje natural o pseudocódigo, realizar una prueba de escritorio y luego implementar el programa. El objetivo principal no es memorizar código, sino aprender a pensar de forma lógica.
